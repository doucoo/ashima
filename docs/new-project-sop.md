# 新项目 SOP

## 目的

规范 `ashima` 项目体系内每一个新项目的创建方式、目录结构、材料归档方式和执行顺序。

## 强制规则

### 1. 每执行一个项目，必须创建一个独立项目文件夹

禁止在没有独立项目目录的情况下直接开始项目工作。

### 2. 项目文件夹命名规则

命名规则必须为：

`project + 编号 + 项目名称`

示例：

- `project001_duet-upgrade`
- `project002_omnigent-compare`
- `project003_global-workbench`

### 3. 所有与该项目相关的内容，全部需要存放到对应的项目文件夹当中

包括但不限于：

- 项目说明
- 调研内容
- 中间草稿
- 截图和附件
- 脚本
- 最终交付物
- 复盘文档

### 4. 不能把项目内容放到电脑的其它位置

禁止把项目资料分散到桌面、下载目录、临时无关文件夹或其它项目目录中。

## 标准流程

### Step 1：开工前阅读

在开始任何新项目之前，必须先阅读：

1. 《全局工作台》
2. 《全局复利与踩坑日志》
3. 《新项目 SOP》

### Step 2：确定项目编号

规则：

- 查当前已有项目编号
- 选择下一个可用编号
- 保证编号递增

### Step 3：创建项目目录

目录名格式：

`projectNNN_项目名称`

### Step 4：初始化目录结构

建议初始化：

```text
projectNNN_项目名称/
├── README.md
├── notes/
├── research/
├── drafts/
├── assets/
├── scripts/
└── deliverables/
```

### Step 5：初始化 README

README 至少包含：

- 项目名称
- 项目目标
- 当前状态
- 负责人/执行者
- 关键里程碑
- 相关文档链接

### Step 6：所有产出都只落在项目目录内

执行过程中新增的任何材料都必须放进该项目目录下的对应子目录里。

### Step 7：收尾与复盘

项目阶段结束后，至少完成：

- 更新项目 README
- 整理 deliverables
- 将高价值经验写入《全局复利与踩坑日志》

## 检查清单

### 新项目创建检查

- [ ] 已阅读三份全局文档
- [ ] 已确定新项目编号
- [ ] 已创建独立项目目录
- [ ] 已初始化基础目录结构
- [ ] 已创建项目 README

### 执行中检查

- [ ] 所有文件都在项目目录内
- [ ] 没有把项目内容散落到其它位置
- [ ] 中间资料已分类归档

### 收尾检查

- [ ] README 已更新
- [ ] 交付物已归档
- [ ] 踩坑与复利经验已补充到全局日志

## 违规处理原则

如果发现以下情况，必须先纠正，再继续执行项目：

- 未创建独立项目目录
- 项目命名不符合规则
- 材料散落在其它位置
- 未阅读三份全局文档就直接开工
